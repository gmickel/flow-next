```

├── skills/            # All 16 skills, patched for Codex

└── hooks.json         # Codex-format Ralph hooks
```

## Agent conversion (agents/*.md → codex/agents/*.toml)

Extract the conversion logic from install-codex.sh. Apply these rules:

**TOML format:**
```toml
# Auto-generated by sync-codex.sh from <name>.md — do not edit manually
name = "<codex_name>"
description = "<from frontmatter>"
model = "<mapped model>"
model_reasoning_effort = "<effort>"   # omit for mini and inherit
sandbox_mode = "<mode>"
nickname_candidates = ["<name1>", "<name2>", "<name3>"]  # scouts only

developer_instructions = """
<body content, backslashes escaped>
"""
```

**Model mapping:**

| Claude Code model | Codex model | Reasoning | Agents |
|-------------------|-------------|-----------|--------|
| opus | gpt-5.4 | high | quality-auditor, flow-gap-analyst, context-scout |
| sonnet (smart scouts) | gpt-5.4 | high | epic-scout, agents-md-scout, docs-gap-scout |
| sonnet (fast scouts) | gpt-5.4-mini | (omit) | build, env, testing, tooling, observability, security, workflow, memory scouts |
| inherit | (omit) | (omit) | worker, plan-sync |

**Sandbox modes:**
- All scouts + quality-auditor + flow-gap-analyst: `sandbox_mode = "read-only"`
- worker + plan-sync: `sandbox_mode = "workspace-write"`

**Nickname candidates** — add 3 per scout for parallel UX. Thematic names.

**Agent body patching:**
- claude-md-scout → agents-md-scout: CLAUDE.md → AGENTS.md, Claude Code → Codex
- Escape backslashes for TOML triple-quoted strings
