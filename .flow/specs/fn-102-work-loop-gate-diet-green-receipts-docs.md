# Work-loop gate diet: green receipts + docs-only gate tiering + event-driven waits

## Goal & Context

The fn-89 pipeline run was trace-mined end to end (2026-07-19 post-mortem): of 2h41 work-tick-to-PR, actual implementation was ~425s (4.4%) while ~50% went to the full unittest suite executing 10 times in one work stage. Every worker runs the suite twice (pre-baseline at `agents/worker.md` "Baseline check" + post-gate "Verify before completing"), each pre-baseline re-proves green on exactly the commit the previous worker's post-gate certified, and the host re-runs tests again at `skills/flow-next-work/phases.md` Phase 4. Worst case: the docs-only task spent 84% of its 55 minutes on suite x4 + smoke x2 - gates its diff could not break - while the REVIEW layer's deterministic triage-skip classified the same diff in 2 seconds.

The fix is mechanical, not semantic: (a) remember, keyed by exact commit hash, that a tree state was already proven green; (b) classify a diff by file path as unable to affect the executable surface. Both are pure plumbing - the flowctl carve-out. This spec deliberately stays outside fn-83 territory: "did this change semantically affect that plan?" remains a judgment question and its deterministic-skip ban stands (decision record `plan-sync-skip-gate-not-viable-2026-07-03`, module phases.md, binds). "Is this byte-identical tree already proven green?" and "does this diff touch only non-executable paths?" involve no judgment.

**Two whitelists, two purposes (design tension resolved):** the review triage-skip (`TRIAGE_ARTIFACT_PREFIXES`) forces REVIEW on `.flow/specs|tasks|epics` because review protects MEANING; the gate tier treats most `.flow/**` as gate-safe because gates protect EXECUTABLES - a spec md cannot break the Python suite. Both layers are correct independently; the spec states this in prose, the evidence strings name their layer, and the gate classifier REUSES the triage helpers (`TRIAGE_DOC_EXTS`, path normalization) so the two lists share primitives rather than drifting.

## Acceptance criteria

- **R1:** Green receipts - baseline reuse. When a full gate (gate_id `unittest` or `smoke`) passes at a worker post-gate OR host Phase 4, `flowctl gate receipt` records it as ONE FILE PER RECEIPT at `.flow/tmp/green-receipts/<sha8>-<gate_id>.json` (`{schema: 1, head_sha, gate_id, timestamp}`, `atomic_write_json`; per-receipt files eliminate the read-modify-write lost-update race - no shared ledger, no lock). A later gate site (worker pre-baseline "Baseline check", host Phase 4) first runs `flowctl gate check --gate <gate_id>`: receipt honored iff receipt `head_sha` == current HEAD AND worktree clean outside `.flow/` (`git status --porcelain` filtered `^.. \.flow/` - the ledger dir is gitignored and must not self-dirty) AND receipt age <= 24h (TTL guards toolchain/env drift on an unchanged tree) AND `schema` matches. ANY failure of any condition (including unparseable/missing/not-a-git-repo) -> exit 1, run the full gate as today. Exit contract: 0 = honored/skip, 1 = run gates, 2+ = error (mirrors triage-skip). Lint/format stay always-run (fast, not receipted, v1). Remote CI gates (land's tri-state, GH Actions) are explicitly OUT of scope - a green receipt never implies CI can be skipped.
- **R2:** Docs-only gate tiering. `flowctl gate classify --base <ref>` classifies the CUMULATIVE task diff (`git diff --name-only <base>...HEAD` + staged/unstaged) at EACH gate decision point (never locked at task start - a fix-loop commit that adds code reclassifies to full on the next gate). Class NON-EXECUTABLE iff EVERY path matches: `docs/**`, `agent_docs/**`, `optimization/**`, `CHANGELOG.md`, `README*`, `GLOSSARY.md`, `STRATEGY.md`, `*.md/*.mdx` under `plugins/flow-next/docs/`, and `.flow/**` EXCEPT `.flow/bin/**` (the deployed CLI copy IS executable surface). NOT gate-safe: `scripts/**`, `plugins/flow-next/scripts/**`, `tests/**`, skill/agent/command `.md` under `plugins/flow-next/{skills,agents,commands}/**` and the codex mirror (sync-codex validates those - they keep the mirror gate), and anything else. Exit codes 0 = docs-only (tier-B), 1 = full gates, 2+ = error (fail-closed to full). Tier-B gate set = sync-codex.sh x2 + guards WHEN the diff touches mirror-source paths (`plugins/flow-next/{skills,agents,commands,references}/**` - the sub-classifier is defined as exactly sync-codex's input tree), else no mirror step; plus lint where repo-configured. Full unittest + smoke are skipped in tier-B. Classifier reuses `_classify_triage_path`'s path-normalization (Windows `\\`->`/`) and `TRIAGE_DOC_EXTS` where semantics align.
- **R3:** Loud skips, machine-parseable. Every honored receipt / tier-B decision writes an evidence line into the task's `tests[]` (fn-99 evidence schema compatible - plain strings) using the prefix convention `GATE_SKIPPED:<gate_id>:<green-receipt <sha8>|docs-only>` plus a human clause, and the worker/host summary carries the same line. Never a silent skip. The two layers stay distinguishable: gate lines say "test gates", review triage receipts stay `mode: triage_skip`.
- **R4:** Event-driven waits. Fixed full-interval sleeps in the work-loop prose are replaced by short-interval bounded until-loops: `references/codex-delegation.md:357-364` (seq 1 6; sleep 10 -> poll every 2s, same 60s bound) and any fixed sleeps in `agents/worker.md`/`phases.md`. Watchdog patterns follow the group-kill lesson (memory `bash-deadline-watchdogs-orphaned-sleep-2026-07-16`).
- **R5:** Scope guard. NO semantic skipping anywhere: predicates are commit-hash equality, worktree cleanliness, receipt age, and path membership ONLY. The fn-83 decision record binds; its record gains a one-line forward-pointer addendum ("fn-102's hash/path-only gate diet is outside this ban - see spec Decision Context"). A regression test asserts the classifier forces FULL gates for every skill/agent/command `.md`, `.flow/bin/**`, and any code extension.
- **R6:** Measurement. After landing, run one docs-only task and one code task through the loop; acceptance: docs-only task runs zero full-suite executions (tier-B evidence lines present), code task's second gate site honors a receipt (GATE_SKIPPED line present), zero green->red regressions attributable to a skipped gate. Record in the task evidence; a follow-up trace post-mortem on the next comparable pilot run targets total gate share <= 50% of the fn-89 baseline.
- **R7:** Cross-platform. flowctl changes land in `plugins/flow-next/scripts/flowctl.py` AND the byte-identical `.flow/bin/flowctl.py` copy (dual-copy invariant). Worker/phases prose edits walk the CLAUDE.md cross-platform checklist: sync-codex.sh x2 idempotent + guards green; canonical prose stays host-portable (no Claude-only constructs; flowctl invocations use the `$FLOWCTL` fallback chain). Classifier is pathlib/normalized (Windows-safe). Worktree-mode workers: `.flow/tmp` is per-checkout, so receipts are naturally scoped per worktree - no cross-worktree reuse (documented, correct: different worktrees have different HEADs).

## Approach

Mirror the review triage-skip architecture one layer down. New flowctl subcommand group `gate` (argparse `add_subparsers` pattern like `memory`/`prospect`): `gate receipt --gate <id>` (write, called after a passing full gate), `gate check --gate <id>` (honor probe, exit-coded), `gate classify --base <ref>` (tier decision, exit-coded). Receipts are per-file (no shared ledger). Worker prose wires three sites: Baseline check (worker.md:52-63) consults `gate check` per gate_id before running; Verify-before-completing (worker.md:364-371) runs full gates per classification then writes receipts; evidence write (worker.md:383-390) carries the GATE_SKIPPED lines. Host phases.md Phase 4 (475-483) consults the same checks. Opt-out: none needed - fail-closed semantics make the feature invisible when receipts/classification don't apply.

## Decision Context

- Post-mortem evidence: fn-89 run, suite x10 = ~4,800s of 9,640s; docs-only task 84% gate time; implementation 4.4%. (memory `session-2026-07-19-fn89-shipped`)
- Per-receipt files over a JSON ledger: eliminates the concurrent read-modify-write lost-update the gap analysis flagged; atomic file create is the only write primitive needed.
- `.flow/bin/**` carve-out: the deployed CLI copy is executable surface - a gate-diet patch must never classify its own live code as docs.
- 24h TTL: hash equality cannot see toolchain/dependency drift on an unchanged tree; a day-scoped receipt bounds that exposure cheaply. No other TTL/pruning in v1 (receipts are tiny; `.flow/tmp` is ephemeral).
- Classification at each gate decision over the cumulative live diff, never locked at task start: a docs-then-code fix-loop must escalate to full gates automatically.
- Review-triage vs gate-tier whitelists: same primitives, different purposes (meaning vs executables) - stated in prose, shared helpers, layer-named evidence.
- fn-83 contrast: that gate was a deterministic proxy for a semantic question and produced a false skip; every predicate here is hash/path/age membership. fn-101's burden-of-proof framing: these belong IN flowctl because no judgment is involved.
- fn-85 sequences AFTER this spec (its scenario matrices baseline the same worker/phases prose); fn-94 touches the same codex-delegation paragraph - coordinate wording at fn-94 plan time.
- Review triage-skip precedent anchors: whitelist constants flowctl.py:26652-26748, `_classify_triage_path` ~26773 (normalization), `_triage_deterministic` 26870-26974 (fail-closed), `cmd_triage_skip` 27125-27310 (receipt shape + exit codes).
